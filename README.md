### AI 模拟面试智能体

设计一个能够自主驱动完整面试流程的智能体，感知求职者的回答内容，主动追问薄弱环节，动态调整提问节奏，并在面试结束后自主生成多维度评估报告，给出改进建议。智能体需体现"感知 → 决策 → 行动 → 反馈"的完整闭环，而不仅是一个简单的问答程序。

参考：构建 3 个智能体，形成面试闭环 。

面试官智能体（Interviewer Agent）：主控流程，根据岗位JD生成开场问题 → 监听候选人回答 → 识别答案薄弱点后追问。需要调用：记忆工具（存储对话历史）、RAG检索工具（从岗位知识库中召回相关面试题）、意图识别工具（判断何时追问/放行）

评估智能体（Evaluation Agent）：在面试结束后拿到全部对话记录，从"专业深度、表达逻辑、应变能力、岗位匹配度"等维度独立打分。需要调用：评分Prompt模板工具、结构化输出工具（生成JSON评估报告）

报告生成智能体（Report Agent）：将评估结果渲染成可读报告，给出改进建议和学习路径。需要调用：Markdown/PDF生成工具

核心难点：面试官智能体的"追问决策"——需要在 Prompt 中设计识别"答案模糊 / 缺乏细节 / 自相矛盾"的规则，并触发追问 Tool Call。
